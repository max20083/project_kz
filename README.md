# Документация проекта K-means

Проект содержит реализацию алгоритма кластеризации **K-means** с нуля (без `sklearn`, `numpy` и других готовых функций кластеризации). Используются только стандартная библиотека Python и, опционально, `matplotlib` для визуализации.

## Структура проекта

| Файл | Назначение |
|------|------------|
| `k_means.py` | Ядро алгоритма — все функции кластеризации |
| `demo.py` | Демонстрация на синтетических данных (150 точек, 3 кластера) |
| `main.py` | Минимальный пример использования на 6 точках |
| `requirements.txt` | Зависимости для графика (`matplotlib`) |

---

## Общая схема алгоритма K-means

```
1. init_centroids()     → выбрать k начальных центроидов
2. assign_clusters()    → назначить каждой точке ближайший центроид
3. update_centroids()   → пересчитать центроиды как средние точек кластера
4. centroids_changed()  → проверить сходимость
5. Повторять шаги 2–4, пока центроиды не стабилизируются
```

Главная функция — `k_means()` — объединяет все шаги в один цикл.

## Файл `k_means.py`

### Типы данных

```python
Point = List[float]      # точка: список координат, например [1.0, 2.5]
Centroid = List[float]   # центроид: такой же формат, что и точка
```

---

### `euclidean_distance(a, b) -> float`

**Назначение:** вычисляет евклидово расстояние между двумя точками.

**Параметры:**
- `a`, `b` — последовательности чисел одинаковой размерности (координаты точек)

**Формула:**

\[
d(a, b) = \sqrt{\sum_{i=1}^{n} (a_i - b_i)^2}
\]

**Как работает:**
1. Для каждой пары координат `(x, y)` вычисляется квадрат разности `(x - y)²`
2. Квараты суммируются
3. Из суммы извлекается квадратный корень

**Пример:**
```python
euclidean_distance([0, 0], [3, 4])  # → 5.0
```

**Используется в:** `assign_clusters`, `centroids_changed`, `inertia`

---

### `mean_point(points) -> Centroid`

**Назначение:** вычисляет «центр масс» — среднее арифметическое набора точек по каждой координате.

**Параметры:**
- `points` — список точек одного кластера

**Возвращает:** одну точку (центроид), координаты которой — средние по оси

**Как работает:**
1. Определяется размерность `dim` (число координат в точке)
2. Создаётся вектор нулей длины `dim`
3. Для каждой точки координаты суммируются
4. Каждая сумма делится на число точек `n`

**Пример:**
```python
mean_point([[0, 0], [2, 4], [4, 0]])  # → [2.0, 1.333...]
```

**Ошибки:** `ValueError`, если список `points` пуст

**Используется в:** `update_centroids`

---

### `assign_clusters(points, centroids) -> List[int]`

**Назначение:** шаг **assignment** — каждой точке назначается метка ближайшего центроида.

**Параметры:**
- `points` — все точки данных
- `centroids` — текущие центроиды (список из `k` точек)

**Возвращает:** список целых меток длины `len(points)`, где `labels[i]` — номер кластера (0 … k−1) для `points[i]`

**Как работает:**
1. Для каждой точки вычисляются расстояния до всех центроидов
2. Выбирается индекс минимального расстояния (`distances.index(min(distances))`)
3. Этот индекс записывается как метка кластера

**Пример:**
```python
points = [[0, 0], [10, 10]]
centroids = [[1, 1], [9, 9]]
assign_clusters(points, centroids)  # → [0, 1]
```

**Используется в:** `k_means`, `demo.main` (проверка согласованности)

---

### `update_centroids(points, labels, k) -> List[Centroid]`

**Назначение:** шаг **update** — пересчёт центроидов после назначения меток.

**Параметры:**
- `points` — все точки данных
- `labels` — текущие метки кластеров
- `k` — число кластеров

**Возвращает:** список из `k` новых центроидов

**Как работает:**
1. Для каждого `cluster_id` от 0 до k−1 отбираются точки с этой меткой
2. Если точки есть — центроид = `mean_point(cluster_points)`
3. Если кластер **опустел** (ни одна точка не назначена) — вместо среднего берётся случайная точка из всего набора данных (защита от «мертвых» центроидов)

**Используется в:** `k_means`

---

### `centroids_changed(old, new, tolerance=1e-6) -> bool`

**Назначение:** проверка сходимости алгоритма — изменились ли центроиды существенно.

**Параметры:**
- `old` — центроиды до обновления
- `new` — центроиды после обновления
- `tolerance` — допустимое расстояние (порог остановки)

**Возвращает:**
- `True` — хотя бы один центроид сдвинулся больше чем на `tolerance` → нужна ещё итерация
- `False` — все центроиды стабилизировались → алгоритм сошёлся

**Как работает:**
1. Попарно сравниваются старые и новые центроиды
2. Для каждой пары вычисляется `euclidean_distance`
3. Если хотя бы одно расстояние > `tolerance`, возвращается `True`

**Используется в:** `k_means`

---

### `init_centroids(points, k, seed=None) -> List[Centroid]`

**Назначение:** случайная инициализация начальных центроидов.

**Параметры:**
- `points` — набор данных
- `k` — число кластеров
- `seed` — необязательное зерно для `random.seed()` (воспроизводимость)

**Возвращает:** `k` точек, выбранных случайно из `points` без повторений

**Как работает:**
1. При переданном `seed` фиксируется генератор случайных чисел
2. Проверяется, что `k <= len(points)`
3. `random.sample(points, k)` выбирает k различных точек
4. Возвращаются **копии** точек (`point[:]`), чтобы изменения центроидов не затрагивали исходные данные

**Ошибки:** `ValueError`, если `k` больше числа точек

**Используется в:** `k_means`

---

### `k_means(points, k, max_iterations=100, tolerance=1e-6, seed=None) -> Tuple`

**Назначение:** главная функция — полный цикл алгоритма K-means.

**Параметры:**
| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `points` | Список точек (списков координат) | — |
| `k` | Число кластеров | — |
| `max_iterations` | Максимум итераций | 100 |
| `tolerance` | Порог сходимости центроидов | 1e-6 |
| `seed` | Зерно для начальной инициализации | None |

**Возвращает:** кортеж `(labels, centroids, iterations)`
- `labels` — метки кластеров для каждой точки
- `centroids` — финальные центроиды
- `iterations` — число выполненных итераций

**Как работает:**
```
centroids ← init_centroids(points, k)
для iteration от 1 до max_iterations:
    labels ← assign_clusters(points, centroids)
    new_centroids ← update_centroids(points, labels, k)
    если NOT centroids_changed(centroids, new_centroids):
        вернуть (labels, new_centroids, iteration)
    centroids ← new_centroids
вернуть (labels, centroids, max_iterations)  # лимит исчерпан
```

**Ошибки:**
- `ValueError`, если `k <= 0`
- `ValueError`, если `points` пуст

**Пример:**
```python
labels, centroids, n = k_means([[1, 2], [5, 8], [1.5, 1.8]], k=2, seed=42)
```

### `inertia(points, labels, centroids) -> float`

**Назначение:** метрика качества кластеризации — **Within-Cluster Sum of Squares (WCSS)**, также называемая *inertia*.

**Параметры:**
- `points` — все точки
- `labels` — метки кластеров
- `centroids` — центроиды

**Формула:**

\[
\text{inertia} = \sum_{i=1}^{N} \| x_i - \mu_{c_i} \|^2
\]

где \(x_i\) — точка, \(c_i\) — её кластер, \(\mu_{c_i}\) — центроид этого кластера.

**Как работает:**
1. Для каждой точки находится её центроид по метке
2. Вычисляется квадрат расстояния до центроида
3. Все квадраты суммируются

**Интерпретация:** чем меньше inertia при фиксированном `k`, тем компактнее кластеры. Используется для сравнения разных запусков или выбора `k` (метод локтя).

**Используется в:** `demo.print_results`

## Файл `demo.py`

Демонстрационный скрипт: генерирует 3 облака точек, запускает K-means, выводит статистику и строит график.

---

### `normal_sample() -> float`

**Назначение:** генерация одного случайного числа из **стандартного нормального распределения** N(0, 1).

**Метод:** преобразование Box-Muller

**Как работает:**
1. `u1`, `u2` — два равномерных случайных числа из (0, 1)
2. `r = sqrt(-2 * ln(u1))`
3. `theta = 2 * pi * u2`
4. Возвращается `r * cos(theta)` — одно нормально распределённое значение

**Зачем:** создавать «облака» точек вокруг центров, а не равномерный шум.

**Используется в:** `generate_clusters`

---

### `generate_clusters(centers, points_per_cluster, spread=1.0, seed=42) -> list`

**Назначение:** синтетический набор данных для демонстрации.

**Параметры:**
| Параметр | Описание |
|----------|----------|
| `centers` | Список истинных центров облаков, например `[(0, 0), (8, 0)]` |
| `points_per_cluster` | Сколько точек генерировать вокруг каждого центра |
| `spread` | Разброс (стандартное отклонение) вокруг центра |
| `seed` | Зерно генератора |

**Возвращает:** плоский список всех точек (все кластеры объединены)

**Как работает:**
1. Фиксируется `random.seed(seed)`
2. Для каждого центра `(cx, cy)` генерируется `points_per_cluster` точек:
   - `x = cx + normal_sample() * spread`
   - `y = cy + normal_sample() * spread`

**Пример (в demo):**
```python
centers = [(0, 0), (8, 0), (4, 7)]
points = generate_clusters(centers, 50, spread=0.8)  # 150 точек
```

### `print_results(points, labels, centroids, iterations) -> None`

**Назначение:** текстовый отчёт о результатах кластеризации в консоль.

**Выводит:**
- общее число точек и кластеров
- число итераций до сходимости
- значение inertia (WCSS)
- размер каждого кластера
- координаты всех центроидов
- первые 10 точек с их метками

**Не возвращает значение** — только печатает.

---

### `try_visualization(points, labels, centroids) -> None`

**Назначение:** построение 2D-графика результата кластеризации.

**Как работает:**
1. Пытается импортировать `matplotlib.pyplot`
2. Если библиотека не установлена — выводит подсказку и завершается
3. Для каждого кластера рисует точки своим цветом (`scatter`)
4. Центроиды — чёрные маркеры «X»
5. Сохраняет файл `kmeans_result.png` (позже работает функция очистки памяти def_temp_files)
6. Вызывает `plt.show()` для отображения окна

**Зависимость:** `matplotlib` (опционально)

---

### `main() -> None`

**Назначение:** точка входа демонстрации.

**Последовательность действий:**
1. Задаёт 3 истинных центра: `(0,0)`, `(8,0)`, `(4,7)`
2. Генерирует 150 точек (3 × 50)
3. Запускает `k_means(points, k=3, seed=42)`
4. Печатает отчёт через `print_results`
5. Проверяет, что метки согласованы с `assign_clusters` (`assert`)
6. Вызывает `try_visualization`

**Запуск:**
```powershell
py demo.py
```

## Файл `main.py`

Минимальный пример без генерации данных и визуализации.

**Что делает:**
1. Импортирует `k_means`
2. Задаёт 6 точек в 2D
3. Запускает кластеризацию с `k=2`
4. Печатает метки, центроиды и число итераций

**Запуск:**
```powershell

```

**Пример вывода:**
```

[[1.17, 1.47], [7.67, 9.0]]  # 2 центроида
3                            # итераций
```
